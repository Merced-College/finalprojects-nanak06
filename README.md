## Program Description
This program simulates non-player characters (NPCs) that make decisions based on their current state, such as hunger, energy, and mood. Each NPC will automatically choose actions like eating, resting, or exploring using a decision-making algorithm.

---

## How to Run
1. Compile the Java files  
2. Run the Main class  
3. Follow the console prompts  

---

## Features
- Create and manage multiple NPCs  
- NPCs make decisions based on their attributes  
- Actions affect NPC state over time  
- Simulation updates continuously  

---

## Data Structures Used
- HashMap → stores NPCs by ID  
- LinkedList → stores NPC list or action history  
- Queue → processes NPC actions in order  

---

## Algorithms Used
- Decision-making algorithm (chooses best action)  
- Search algorithm (find NPC by ID)  
- State update algorithm (changes NPC attributes over time)
